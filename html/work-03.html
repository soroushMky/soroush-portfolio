01 Overview

Nomi is a personal finance analytics platform that transforms raw bank transaction data into actionable insights. Built as a portfolio project to demonstrate proficiency in data engineering, machine learning, SQL, and AI integration — the exact skills required for modern data analyst and AI engineering roles.

The app processes a real Kaggle personal finance dataset — 617 transactions across 2018–2019 — and surfaces patterns, anomalies, and trends through interactive visualizations and an AI coach powered by Anthropic's Claude.

02 Architecture

A clean four-layer system: a vanilla JS frontend talks to a Flask REST API, which reads from SQLite through a SQLAlchemy ORM and calls out to Claude for anything requiring reasoning over the user's full financial context.

03 Data Pipeline

CSV upload supports multiple bank formats — TD Bank, RBC, BMO, Chase, and more — with flexible column mapping to handle inconsistent schemas. Data cleaning automatically strips credit card payments, paychecks, and non-debit entries before anything touches the database.

Raw CSV (bank export) ↓ pandas ingestion + column mapping ↓ Data cleaning (remove non-spending rows) ↓ SQLite storage via SQLAlchemy ORM ↓ Isolation Forest anomaly detection ↓ Health score calculation (5 metrics) ↓ REST API endpoints ↓ Chart.js visualizations + AI insights
04 Data Analytics

An interactive dashboard surfaces KPI cards with animated number transitions — total spend, monthly averages, top categories, anomaly counts. A monthly trend line chart caught a real home renovation spike (+82% in May–June 2018), while a horizontal bar chart breaks spending across 18 categories. Budget-vs-actual and year-over-year comparisons round out the analytics layer.

05 AI & Machine Learning

An Isolation Forest model flags transactions exceeding 2× the category average, automatically filtering out recurring payments to avoid false positives. Every flagged anomaly is then passed to Claude, which explains why it stands out in plain language — the model detects, the LLM explains.

# Simplified anomaly detection pipeline from sklearn.ensemble import IsolationForest model = IsolationForest(contamination=0.02, random_state=42) model.fit(transaction_features) anomaly_scores = model.decision_function(transaction_features) flagged = transactions[model.predict(transaction_features) == -1] # Pass flagged transactions to Claude for explanation explanation = claude_client.explain(flagged_transaction)

Beyond anomaly detection, Claude powers a full AI Financial Coach chat interface with complete context awareness, a one-click AI Report Generator that produces an executive summary with monthly analysis and personalized recommendations, and feeds into a composite Financial Health Score (0–100) built from spending consistency, budget adherence, anomaly frequency, and goal progress.

06 SQL & Database

Five real SQL queries run live against SQLite, each with a toggleable code view so the query itself is part of the presentation, not hidden behind an API call: top merchants by spend, spending by account with averages, biggest single transactions, category breakdown with calculated percentages, and a monthly comparison built on strftime grouping.

07 Key Technical Decisions

Why Isolation Forest for anomaly detection? It's unsupervised — no labeled training data required, which is ideal for personal finance where "normal" varies entirely by user. It isolates anomalies by randomly partitioning the feature space, making it efficient even on small datasets.

Why SQLite instead of PostgreSQL? Zero-configuration and sufficient for single-user financial data. The SQLAlchemy ORM means switching to PostgreSQL for production requires only a one-line config change — demonstrating production-aware architecture without over-engineering the prototype.

Why Flask over Django? Flask's minimal footprint gives explicit control over every component — ideal for a portfolio project where understanding the full stack matters more than rapid scaffolding.

Why Claude over GPT? Claude's larger context window allows injecting the full transaction history, monthly totals, and category breakdowns into every AI request — enabling genuinely personalized financial advice rather than generic responses.

08 Dataset

Sourced from a public Kaggle personal finance dataset. Raw data arrived as 806 transactions across 2018–2019; after cleaning — removing credit card payments, paychecks, and non-debit entries — 617 spending transactions remained across 18 categories and 3 accounts (Checking, Platinum Card, Silver Card).

The standout finding the model surfaced on its own: a home renovation in May–June 2018 caused an 82% spending spike — $17,200 to a single contractor, Mike's Construction Co.
